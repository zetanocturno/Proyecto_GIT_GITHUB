# Proyecto_GIT_GITHUB
Trabajo final diplomado de uso de Git y GitHub

## Objetivo
Gestionar el ciclo de vida de un proyecto de arquitectura de software mediante el uso
de ramas, seguimiento de tareas y documentación técnica en un repositorio remoto.
## Contexto
El reto consiste en simular el entorno de trabajo de un arquitecto de software que debe
organizar la documentación técnica de un nuevo sistema. No se requiere programar la
lógica, sino utilizar GitHub para administrar la creación de los "contratos"
(documentación de la API) y el diseño del sistema, asegurando que cada avance esté
registrado de forma profesional y organizada antes de la fecha de entrega de la
práctica.
## Instrucciones
1. Inicialización: Crear un repositorio público en GitHub con un archivo
README.md inicial.
2. Planificación con Issues: Crear al menos 3 Issues que representen las tareas del
proyecto (ej. "Diseño de endpoints REST", "Definición de esquema GraphQL",
"Diagrama de arquitectura").
3. Flujo de Trabajo (Branches): Crear una rama independiente para cada Issue (ej.
feat/rest-api). No se permite trabajar directamente en la rama main.
4. Uso de IA y documentación: Utilizar herramientas de IA para generar las
descripciones técnicas de los archivos. Los commits deben registrarse con
fechas anteriores a la entrega de la práctica.
5. Integración (Pull Requests): Subir los cambios de cada rama al repositorio
mediante un Pull Request (PR), vinculándolo al Issue correspondiente y
realizando el merge a la rama principal tras una breve descripción de los
cambios
